# Kanban 交易系统 PRD

> **版本**: v3.0 | **更新日期**: 2026-04-28

---

## 1. 项目背景

### 1.1 问题

**交易员工作流痛点：**
- 新闻、市场数据、警报分散在不同平台，需要手动切换
- 市场扫描结果需要人工肉眼筛选，效率低下
- 不同数据源（加密货币、A股、美股、港股）分散在不同平台
- 缺乏从"粗筛 → 精筛 → 入场分析"的完整工作流支撑

**当前已有基础设施：**
- `quant-core` 统一数据基础设施已部署
- `/api/scan/market` API 已上线（Scanner 粗筛）
- `/api/scan/three-screen` API 已上线（异步三滤网精筛）
- 三滤网交易法信号生成器已实现

### 1.2 目标

**核心目标：**
- 实现从新闻 → 警报 → 市场扫描 → 三滤网精筛 → 多周期共振的完整交易工作流
- 提供多种市场扫描策略（交易量突破、布林带、趋势、连续K线、多周期变化）
- 支持三滤网信号精筛，过滤假信号
- 提供浮动控件实现品种快速搜索与管理

**成功指标：**
- 扫描响应时间 < 30秒（单交易所，50个品种）
- 三滤网信号确认准确率 > 70%（通过历史回测验证）

## 2. 用户故事

### US-01：浏览新闻事件

- **角色**：交易员
- **场景**：当刚打开 Kanban，想要了解最近的市场动态和新闻报道时
- **意图**：我希望快速浏览最新的新闻事件列表
- **动机**：以便掌握市场情绪和近期热点

### US-02：监控 TradingView 警报

- **角色**：交易员
- **场景**：当在 TradingView 设置了技术指标警报，需要统一查看触发情况时
- **意图**：我希望有一个警报中心，集中展示所有预设警报的状态和数据
- **动机**：以便无需切换窗口即可监控所有警报触发情况

### US-03：市场扫描 - 粗筛候选品种

- **角色**：交易员
- **场景**：当需要从大量标的中快速找到符合条件的候选品种时
- **意图**：我希望选择交易所和扫描策略（放量突破、布林带等），一键扫描市场
- **动机**：以便从海量市场数据中快速筛选出潜在交易机会

### US-04：三重滤网 - 精筛候选品种

- **角色**：交易员
- **场景**：当市场扫描返回一批候选品种，需要进一步过滤假信号时
- **意图**：我希望使用三滤网交易法（M30趋势确认 + M5回调识别 + M1入场点）对候选品种进行精细筛选
- **动机**：以便过滤假突破，提高信号质量，只保留高概率交易机会

### US-05：多周期共振 - 精细分析

- **角色**：交易员
- **场景**：当在三重滤网筛选出候选品种后，需要进一步验证入场时机时
- **意图**：我希望看到各周期（M1/M5/M30/4H/1D）的 K 线图表、趋势共振评分、入场价、止损、盈亏比
- **动机**：以便最终确认交易决策，执行下单

### US-06：品种搜索与常用品种管理

- **角色**：交易员
- **场景**：当在任何页面想要快速添加一个品种到常用品种列表时
- **意图**：我希望有一个浮动控件，支持搜索品种库并一键添加到常用品种
- **动机**：以便快速管理常用品种，无需专门进入某个页面操作

## 4. 功能说明

### F-01：新闻事件中心

#### F-01-1：新闻列表展示
展示最近的市场新闻事件列表，按时间倒序排列。

#### F-01-2：新闻详情
点击新闻标题展开详情，支持标记已读/未读状态。

---

### F-02：警报中心

#### F-02-1：TradingView 警报数据获取
通过 Chrome CDP 连接 TradingView，扫描所有打开的图表布局，提取指标数值和警报状态。

#### F-02-2：警报列表展示
展示所有警报的触发时间、品种、指标名称、当前值、触发条件。

---

### F-03：市场扫描

#### F-03-1：扫描策略选择
支持 5 种扫描策略：
- `volume_breakout`：交易量突破（放量倍数 > 2x，价格变化 > 3%）
- `bollinger`：布林带分析（价格触及上下轨）
- `trending`：趋势分析（均线斜率）
- `consecutive`：连续K线（连续 3 根同向K线）
- `multi_changes`：多周期变化（跨周期信号确认）

#### F-03-2：交易所选择
支持多交易所扫描：OKX（加密）、SSE（上交所）、SZSE（深交所）。

#### F-03-3：扫描参数配置
- 时间周期：5m / 15m / 30m / 1h / 4h
- 放量倍数阈值：1.5 ~ 5.0
- 最小价格变化：1% ~ 10%
- 返回数量限制：5 ~ 50

#### F-03-4：扫描结果展示
展示品种/方向/强度/放量/涨跌/M30趋势/回调/入场/建议等字段。

---

### F-04：三重滤网

#### F-04-1：品种选择
- 常用品种：从 instruments.yaml 加载，支持批量选择
- 交易所补充：从 quant_core/data/symbols/ 加载对应品种

#### F-04-2：三滤网参数配置
- M30 周期：5m / 15m / 30m / 1h / 4h
- M5 周期：5m / 15m / 30m
- M1 周期：1m / 5m / 15m
- K 线数量：30 ~ 120

#### F-04-3：异步扫描流程
1. POST /api/scan/three-screen 提交任务，获取 task_id
2. 轮询 GET /api/scan/three-screen/{task_id} 查询状态
3. 进度条显示扫描进度
4. 完成后展示做多/做空机会列表

#### F-04-4：三滤网结果展示
- 做多机会：展示趋势、回调、M1信号、强度、支撑/压力、入场价、止损、盈亏比
- 做空机会：同上

---

### F-05：多周期共振

#### F-05-1：多周期 K 线图表展示
通过 lightweight-charts 渲染 K 线图，支持周期切换（1m/5m/30m/4h/1D）。

#### F-05-2：三滤网入场信号
M30 趋势（MomentumFilter）+ M5 回调（PullbackFilter）+ M1 入场（EntryFilter），计算信号强度 0-100，展示入场价、止损、盈亏比。

#### F-05-3：共振评分仪表盘
计算各周期趋势一致性，评分 0-100，显示上涨/下跌/震荡分布。

#### F-05-4：交易建议
根据共振评分和矛盾检测生成建议：强烈买入/买入/观望/卖出/强烈卖出，含置信度和原因说明。

---

### F-06：品种搜索浮动控件

#### F-06-1：品种库数据加载
从 quant_core/data/symbols/ 目录加载各交易所品种列表：
- okx.txt：OKX 加密货币
- sse.txt：上海证券交易所 A 股
- szse.txt：深圳证券交易所 A 股
- nasdaq.txt / nyse.txt：美股
- hkex.txt：港股

#### F-06-2：模糊搜索
支持按品种代码或名称模糊搜索，返回匹配结果。

#### F-06-3：添加到常用品种
选择品种后，添加到 instruments.yaml 的 instruments 列表中，包含：
- symbol：品种代码
- name：品种名称（用户输入）
- exchange：交易所
- source：数据源

#### F-06-4：浮动控件 UI
- 右下角悬浮按钮（🔍 图标）
- 点击展开搜索面板
- 包含交易所筛选、搜索框、结果列表、添加按钮

## 5. API 接口

### 5.1 市场扫描 API

**端点**：POST /api/scan/market

| 参数 | 类型 | 说明 |
|------|------|------|
| scanners | List[str] | 扫描类型列表 |
| exchanges | List[str] | 市场列表 [okx/sse/szse] |
| use_signal_filter | bool | 是否启用三滤网精筛 |
| timeframe | str | 时间周期 (5m/15m/30m/1h/4h) |
| volume_multiplier | float | 放量倍数阈值 |
| price_change_min | float | 最小价格变化百分比 |
| limit | int | 返回结果数量上限 |

### 5.2 三滤网扫描 API

**端点**：POST /api/scan/three-screen?source=okx

**请求体**：
```json
{
    "symbols": ["BTC-USDT", "ETH-USDT"],
    "exchanges": [],
    "bar_size_m30": "30m",
    "bar_size_m5": "5m",
    "bar_size_m1": "1m",
    "num_m30": 60,
    "num_m5": 60,
    "num_m1": 60
}
```

**异步查询**：GET /api/scan/three-screen/{task_id}

## 6. 技术架构

### 6.1 系统架构

- **前端**：Streamlit 多页面应用，部署于本地 Mac（port 8502）
- **后端**：quant-core FastAPI 服务，远程部署于 100.82.238.11:8005
- **数据源**：OKX / TradingView / IB / TdxQuant 通过 quant-core 统一接入

### 6.2 核心模块

| 模块 | 说明 |
|------|------|
| quant_core/scanner | 市场扫描模块：volume_breakout/bollinger/trending/consecutive/multi_changes |
| quant_core/factors | 技术因子模块：MACD/KDJ/RSI/均线/斐波那契/三滤网信号 |
| quant_core/data | 市场数据模块：交易所配置/品种列表/screener映射 |
| trading/kanban | Streamlit 前端：多周期共振/市场扫描/三重滤网/警报监控/新闻 |

### 6.3 页面文件结构

```

├── app.py                    # 主入口，页面导航，浮动控件
├── instruments.yaml          # 常用品种配置
├── pages/

│   ├── 1_alerts.py          # 警报中心
│   ├── 2_market_scan.py     # 市场扫描
│   ├── 3_three_screen.py    # 三重滤网
│   └── 4_resonance.py       # 多周期共振
└── src/
    ├── config.py             # 配置
    ├── data.py               # 数据获取
    └── analysis.py           # 共振计算
```
